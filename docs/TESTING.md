# テスト方針

## 自動テスト

Node.js標準の `node:test` と `assert` のみを使う。外部テストフレームワークは導入しない。

### 採点

- 譜面が50ノート固定で重複IDを持たない
- 最大点が100
- PERFECT／GOODの境界値を含む
- 未入力がMISSになる
- 判定窓外の近接入力がMISSとして1ノートを消費する
- 遠い空振りが固定譜面を変化させない
- 50回の完全入力が100点になる

### 素材

- BGMとSEがPCM WAVとして妥当
- BGMが60〜70秒
- SEが2.3〜2.5秒
- 容量がGitHub Pages向けの上限内
- スプライトが1536 × 1024のRGBA PNG

### セキュリティ

- CSPが存在する
- インライン／外部スクリプトがない
- npm依存がない
- 秘密情報パターンが `.gitignore` に含まれる
- 永続ストレージと動的コード評価を使わない

### ビルド

- 配布に必要なHTML、CSS、JS、画像、音源、`.nojekyll` が存在する
- 中間画像を配布しない
- 配布HTMLにCSPが残る

## Dockerスモークテスト

CIで毎回イメージをビルドし、次を確認する。

1. 最終イメージの実行ユーザーがUID 101
2. コンテナが8080番で起動
3. `/` へのHEADリクエストが成功

## 手動確認項目

- デスクトップでスペースとクリックが同じ判定になる
- iOS／Android相当幅でタップでき、横スクロールしない
- 鳥が開始時に飛来・着地し、通常／失敗／ジャンプ／終了モーションを表示する
- 鳥声の最後とジャンプ判定が一致する
- タブを隠すと停止し、戻るボタンで再開する
- Xボタンが得点と現在URLを設定した投稿画面を別タブで開く
- リロード後に前回の得点が残らない
