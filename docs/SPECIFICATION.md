# ゲーム仕様

## 1. 概要

「トリ・ステップ・パレード」は、群れの鳥と同じタイミングで1つの入力を行うブラウザ向けリズムゲームである。画面を見るだけでも遊べるが、判定基準はWeb Audioの再生クロックであり、音を聴いて遊ぶことを主とする。

## 2. 1プレイの構成

| 項目 | 値 |
| --- | ---: |
| BGM | 66.0秒、100 BPM、4/4拍子 |
| 判定数 | 50 |
| 最初の判定 | 3.0秒 |
| 判定間隔 | 1.2秒（2拍） |
| 最後の判定 | 61.8秒 |
| 最大点 | 100点 |

イントロとカウントの後、強拍に合わせて50回入力する。8、16、24、32、40、48番目の判定はジャンプであり、2.4秒前から鳥声SEが鳴る。最後の「ビャー！！」に重なる入力で群れと一緒に飛ぶ。

## 3. 入力

- キーボード: `Space`
- マウス／トラックパッド: ゲーム画面の押下
- タッチ端末: ゲーム画面のタップ

マウスとタッチは `pointerdown` に統一する。300ms以内の近い未判定ノートへ割り当て、±150msを超える場合はそのノートをMISSとして消費する。300msより離れた空振りは画面にMISSを表示するが、50個の採点対象は消費しない。

## 4. 採点

入力時刻と対象ノート時刻の差を `offset = input - target` とする。負値は早い、正値は遅い。

| 判定 | 時間差 | 点 |
| --- | ---: | ---: |
| PERFECT | `|offset| <= 70ms` | 2 |
| GOOD | `70ms < |offset| <= 150ms` | 1 |
| MISS | `|offset| > 150ms` または未入力 | 0 |

50ノート × 2点で最大100点。正規化や小数点丸めは行わない。

## 5. モーション

- ゲーム開始: 飛来 → 着地
- 通常入力: 左足／右足のステップを交互に表示
- PERFECT: 胸を張るポーズと小さなポップ
- GOOD: 控えめなステップ
- MISS: 驚いてよろける
- ジャンプ合図: しゃがむ
- ジャンプ成功: 飛翔 → 着地
- 終了: フィニッシュポーズ → 群れで飛び去る

周囲の鳥は100 BPMの拍ごとに左右の足を切り替える。プレイヤーの鳥は入力結果で動くため、成功時は群れと同期し、失敗時だけ明確に動きが崩れる。

## 6. タイミングモニター

画面下段に直近16件を表示する。中央がジャスト、左が早い、右が遅い。±70msの領域を緑、±150msまでを黄、MISSを赤で示す。未入力MISSは右端に表示する。

## 7. 終了と共有

終了ダイアログには得点、PERFECT／GOOD／MISS数、得点帯メッセージを表示する。「X 結果をポスト」は、得点と内訳、ハッシュタグ、現在のページURLをX Web Intentへ渡す。自動投稿は行わず、最終送信はX上でユーザーが行う。

## 8. データ

セッション状態はJavaScriptのメモリ上だけに保持する。Cookie、Local Storage、Session Storage、IndexedDB、サーバー送信を使用しない。リロード時に結果は消える。

## 9. 一時停止

プレイ中にタブが非表示になるとAudioContextとアニメーションを停止する。復帰時は明示的な「ゲームに戻る」操作で再開する。バックグラウンド制限による大量MISSを避けるためである。

## 10. 対応方針

- 320px以上の画面幅
- Pointer Events、Web Audio API、ES2022相当を実装した現行ブラウザ
- キーボードフォーカス、`aria-live`、判定の色以外の文字表示に対応
- `prefers-reduced-motion` ではアニメーション時間を短縮
